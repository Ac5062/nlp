Project Details

This project develops an end-to-end NLP pipeline that automatically evaluates the quality of user-generated questions, classifying them into three tiers — Low, Medium, and High quality — using the Stack Overflow community as a real-world benchmark.

The system combines TF-IDF vectorization with a rich set of handcrafted linguistic features (word count, code presence, sentence structure, URL indicators, capitalization ratios) and trains a Logistic Regression classifier with balanced class weights to handle the inherent skew in question quality distributions.

A Streamlit web application provides a real-time inference interface where users can paste any question and instantly receive a quality score with class probabilities — making this tool practical for platforms that want to guide users toward writing better questions before submission.

The improved v2 model achieved 85.6% accuracy and 0.956 ROC AUC, representing a significant jump from the baseline v1 (65.5% accuracy), demonstrating the impact of enhanced feature engineering and model tuning.

Dataset

The project uses the 60k Stack Overflow Questions with Quality Rating dataset available on Kaggle (kaggle.com/datasets/imoore/60k-stack-overflow-questions-with-quality-rate). It contains 60,000 questions with community-assigned quality labels across three categories:

Label Quality Tier Definition Count
HQ High Quality Score > 30, never edited by community ~20,000
LQ_EDIT Medium Quality Low vote score, community-edited but still open ~20,000
LQ_CLOSE Low Quality Closed by community vote ~20,000

Each record contains a Title, Body (HTML), creation date, and quality label Y. The dataset is balanced across classes. A 75/25 train-test split is used, yielding ~45K training samples and ~15K test samples.

Overall Gap in Use Case & How to Solve with NLP Models

While the current pipeline achieves strong performance, several critical gaps limit its real-world applicability. The following analysis identifies these limitations and maps them to advanced NLP solutions:

🔴 Current Limitations

  • TF-IDF loses word-order and semantic context entirely
  • No understanding of domain-specific technical terminology
  • Low / Medium quality boundary is inherently fuzzy (65% recall on LQ_CLOSE in v1)
  • Static model — cannot adapt to evolving language patterns
  • No sentence-level or paragraph-level structure awareness
  • Handcrafted features are brittle and require manual curation
  • Cannot explain why a question is low quality (no interpretability)
  • Code blocks stripped — important technical signal is discarded

🟢 NLP Model Solutions

  • BERT / RoBERTa: Contextual embeddings capture word meaning in sentence context
  • CodeBERT: Jointly trained on code + NL — ideal for SO questions with code snippets
  • Transformer fine-tuning: Domain-adapt on SO corpus for technical vocabulary
  • Contrastive Learning: Pull similar-quality questions together in embedding space
  • Seq2Seq feedback model: Generate natural-language explanations of quality issues
  • Active Learning: Re-train on ambiguous boundary samples to sharpen decision boundaries
  • Attention visualization: Highlight which tokens drove the quality prediction
  • Multi-task learning: Jointly predict quality + topic tag for richer signal

The most impactful next step would be fine-tuning CodeBERT or StackBERT on this dataset — expected to push accuracy to 90%+ while simultaneously providing sentence-level interpretability through attention weights. This would transform the tool from a classifier into a quality coach that explains exactly what to fix.
